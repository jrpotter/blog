    Bookshelf.LTuple.Basic #

    The following is a representation of a (possibly empty) left-biased tuple. A left-biased n-tuple is defined recursively as follows:

    ⟨x₁, ..., xₙ⟩ = ⟨⟨x₁, ..., xₙ₋₁⟩, xₙ⟩

    Note a Tuple exists in Lean already. This implementation differs in two notable ways:

    1. It is left-associative. The built-in Tuple instance evaluates e.g. (x₁, x₂, x₃) as (x₁, (x₂, x₃)) instead of ((x₁, x₂), x₃).
    2. Internally, the built-in Tuple instance is syntactic sugar for nested Prod instances. Unlike this implementation, an LTuple is a homogeneous collection.

    In general, prefer using Prod over LTuple. This exists primarily to solve certain theorems outlined in [^1].

    [^1]: Enderton, Herbert B. A Mathematical Introduction to Logic. 2nd ed. San Diego: Harcourt/Academic Press, 2001.
